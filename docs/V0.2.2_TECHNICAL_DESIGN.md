## New API Endpoints (added to api.ts)

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/api/sprints/{id}/start` | Trigger orchestrator |
| POST | `/api/sprints/{id}/pause` | Pause execution |
| POST | `/api/sprints/{id}/resume` | Resume from pause |
| POST | `/api/sprints/{id}/cancel` | Graceful cancel |
| POST | `/api/sprints/{id}/message` | Mid-sprint instruction |

All wrapped in try/catch вҖ” gracefully handles backend not implementing these yet.

## Components

### Sprint Launcher (`SprintLauncher.tsx`)
- Goal textarea with Ctrl+Enter shortcut
- Advanced options (collapsible): model override, budget, parallel tasks
- Creates sprint вҶ’ attempts start вҶ’ calls `onSprintStarted(sprintId)`

### Command Center (`CommandCenter.tsx`)
- Orchestrates all sub-components
- Connects WebSocket on mount, disconnects on unmount
- Keyboard shortcut handler for A/R approval
- Timer display (elapsed seconds)
- Sprint controls (pause/resume/cancel)

### Activity Feed (`ActivityFeed.tsx`)
- Renders WSEvent[] as FeedEvent components
- Auto-scroll to bottom with manual override
- "Scroll to latest" FAB when scrolled up
- Agent filter support

### Feed Event (`FeedEvent.tsx`)
- Switch-based rendering for event types:
  - `agent.thinking` вҖ” muted italic
  - `agent.message` вҖ” chat bubble with avatar
  - `agent.action` вҖ” file icon + path
  - `task.completed` вҖ” green card
  - `task.failed` вҖ” red card
  - `approval.requested` вҖ” yellow card with approve/reject buttons
  - `budget.warning` вҖ” yellow text
  - `sprint.completed` вҖ” celebration card
  - Default вҖ” generic JSON display

### Other Components
- **ProgressBar** вҖ” animated bar with task count
- **TeamStatus** вҖ” agent list with status dots, click to filter feed
- **TaskList** вҖ” expandable task cards with acceptance criteria
- **CostTracker** вҖ” total + per-agent bar chart
- **SprintChat** вҖ” input with agent targeting dropdown
- **ArtifactDrawer** вҖ” collapsible bottom panel with file list + preview

## WebSocket Integration

Uses existing `SprintWebSocket` class from `src/lib/ws.ts`:
- Auto-reconnect with exponential backoff
- Sequence tracking for catch-up
- Events accumulated in `useSprintFeed` Zustand store
