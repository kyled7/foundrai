# FoundrAI v0.2 — UI-First Platform

> **Date:** 2026-03-05
> **Status:** Proposed
> **Prerequisite:** Phase 0–4 Complete (CLI engine, API, WebSocket, plugins, templates, multi-team)
> **Goal:** Move all control from CLI to Web UI — users should never need to open a terminal to use FoundrAI

---

## Executive Summary

FoundrAI v0.1 (Phase 0–4) shipped a powerful engine: orchestrator, 17 API routes, WebSocket, plugin system, templates, and multi-team coordination. But everything still starts from CLI (`foundrai init`, `foundrai sprint start`). The current UI is a viewer — it displays but doesn't control.

**v0.2 inverts the model:** UI becomes the primary interface. CLI becomes a power-user alternative.

### v0.2 Definition of Done
A new user can:
1. Open browser → create project → configure team → start sprint → approve/reject → view results
2. **Never open a terminal**
3. Experience realtime, interactive, polished UI

---

## Tech Stack

| Layer | Technology | Rationale |
|-------|-----------|-----------|
| UI Framework | React 18 + TypeScript | Already in use, keep consistent |
| Build | Vite 5 | Already in use, keep consistent |
| Styling | Tailwind CSS 4 + shadcn/ui | High-quality component library, built-in dark mode |
| State | Zustand | Lightweight, simple, sufficient for this app |
| Data Fetching | TanStack Query v5 | Cache, refetch, optimistic updates |
| Routing | TanStack Router | Type-safe, file-based routing |
| Charts | Recharts | Lightweight, React-native charting |
| WebSocket | Native WS + reconnect logic | Realtime sprint events |
| Code Viewer | @uiw/react-codemirror | Syntax highlighting for artifacts |
| Icons | Lucide React | Consistent, tree-shakeable |

---

## Information Architecture

```
/                           → Dashboard (project list, overview stats)
/projects/new               → Project Creation Wizard (3 steps)
/projects/:id               → Project Detail (sprints, team, settings)
/projects/:id/sprint        → Sprint Command Center ⭐ (core experience)
/projects/:id/sprint/:sid   → Sprint Detail / Replay
/projects/:id/analytics     → Analytics & Cost
/projects/:id/team          → Team Config
/templates                  → Template Browser / Marketplace
/settings                   → Global Settings (API keys, models, sandbox)
```

### Phase 0.2.2 — Sprint Command Center ⭐
**Goal:** Core interactive sprint experience — the "wow" page

#### Tasks

##### Backend Prerequisites

- [ ] **v0.2.2-01** — Sprint lifecycle API
  - `POST /api/sprints/{id}/start` — trigger orchestrator (run in background thread/process)
  - `POST /api/sprints/{id}/pause` — pause orchestrator loop
  - `POST /api/sprints/{id}/resume` — resume from pause
  - `POST /api/sprints/{id}/cancel` — graceful cancel (finish current task, then stop)
  - Response: sprint status + ETA

- [ ] **v0.2.2-02** — Mid-sprint communication API
  - `POST /api/sprints/{id}/message` — user sends instruction to team
  - Body: `{ message: string, target_agent?: string }`
  - Injected into orchestrator's message bus as "human_instruction" event
  - Agents pick up and factor into their next decisions

- [ ] **v0.2.2-03** — WebSocket enhancement
  - Hook into orchestrator's event bus (register WS broadcast callback)
  - Event types to broadcast:
    ```
    sprint.started, sprint.paused, sprint.completed, sprint.failed
    task.created, task.started, task.completed, task.failed
    agent.thinking, agent.action, agent.message
    artifact.created, artifact.updated
    approval.requested, approval.resolved
    budget.warning, budget.exceeded
    ```
  - Include sequence numbers for client catch-up on reconnect
  - Reconnect protocol: client sends `{ type: "catch-up", lastSequence: N }`

##### Frontend

- [ ] **v0.2.2-04** — Sprint launcher component
  - Large text input: "Describe what you want to build..."
  - Recent goals dropdown (re-run similar sprint)
  - Advanced options (collapsible): override model, budget, parallel tasks
  - "Start Sprint" button → `POST create_sprint` → `POST start` → redirect to Command Center
  - Keyboard shortcut: Ctrl+Enter to start

- [ ] **v0.2.2-05** — Sprint Command Center layout (`/projects/:id/sprint`)
  - **Two-column layout** (responsive → single column on mobile)
  - Left panel (30%): Sprint info, progress, team status, task list
  - Right panel (70%): Live activity feed
  - Bottom drawer: Artifact viewer (expandable)
  - Top bar: Sprint goal, status badge, timer, cost counter, control buttons (Pause/Resume/Cancel)

- [ ] **v0.2.2-06** — Left panel components
  - **Progress bar:** animated, percentage based on tasks completed/total
  - **Team Status:** agent avatars with status dots (idle/working/waiting/done)
    - Click agent → filter feed to show only that agent's events
  - **Task List:** compact list with status icons
    - Click task → expand inline: description, acceptance criteria, assigned agent, result
    - Status flow: `⏳ pending → 🔄 in_progress → ✅ done / ❌ failed`
  - **Cost tracker:** live updating from WebSocket events, bar chart per agent

- [ ] **v0.2.2-07** — Activity Feed (right panel)
  - Scrolling timeline of events, newest at bottom (chat-like)
  - Event types with distinct styling:
    - 🤖 **Agent thinking** — italic, muted, collapsible reasoning chain
    - 💬 **Agent message** — chat bubble with agent avatar
    - 📝 **Action** — "Created file: src/auth.ts" with file icon
    - ✅ **Task complete** — green card with summary
    - ⚠️ **Approval needed** — highlighted card with Approve/Reject buttons
    - 💰 **Budget warning** — yellow alert
    - 👤 **User message** — right-aligned bubble (from mid-sprint chat)
  - Auto-scroll to bottom (with "scroll to latest" button when scrolled up)
  - Filter bar: All | PM | Dev | QA | Approvals only

- [ ] **v0.2.2-08** — Inline approval system
  - Approval card in feed: agent name, action description, context
  - "Approve" (green) / "Reject" (red) buttons + optional comment field
  - Keyboard: `A` to approve, `R` to reject (when approval is focused)
  - Toast notification when new approval arrives
  - Sound notification (optional, configurable in settings)
  - Badge count on sidebar: "2 pending approvals"

- [ ] **v0.2.2-09** — Mid-sprint chat
  - Input field at bottom of feed: "Send instruction to team..."
  - Dropdown to target specific agent or broadcast to all
  - Messages appear in feed as user bubbles
  - Quick actions: "Prioritize error handling" / "Skip tests for now" / "Focus on [task]"

- [ ] **v0.2.2-10** — Artifact viewer (bottom drawer)
  - List of created/modified files with type icons
  - Click file → CodeMirror viewer with syntax highlighting
  - Diff view toggle: see changes vs original (if file existed)
  - "Copy to clipboard" / "Open in editor" buttons
  - File tree view (collapsible) for multi-file artifacts
  - Image preview for non-code artifacts

- [ ] **v0.2.2-11** — Sprint completion flow

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|-----------|
| WebSocket reliability | Sprint feels laggy/broken | Reconnect + catch-up protocol, sequence numbers |
| Orchestrator blocking | Can't start sprint from API | Run orchestrator in background thread/process with event bus |
| Large activity feeds | Performance degrades | Virtual scrolling (TanStack Virtual), event pagination |
| File system access from web | Can't pick project directory | Text input + validation first, Tauri native dialog later |
| Scope creep | Never ships | Phase 0.2.2 (Command Center) is MVP — ship that first |

## Migration Path

1. **v0.2 frontend** lives in `frontend-v2/` alongside the current `frontend/`
2. Backend serves both: existing `frontend/dist/` at `/v1`, new build at `/`
