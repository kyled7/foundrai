# FoundrAI v0.2.3 — Technical Design: Analytics & Insights

> **Date:** 2026-03-05
> **Status:** Draft
> **Goal:** Full analytics dashboard with cost/performance charts, global stats, sprint replay with timeline scrubber

---

## Architecture Overview

Three deliverables:

1. **Project Analytics** — Replace placeholder at `/projects/$projectId/analytics` with charts and metrics
2. **Global Analytics** — Enhance Dashboard (`/`) with aggregate stats across all projects
3. **Sprint Replay** — Playback past sprints with timeline scrubber and activity feed

**Charts library:** Recharts (add `recharts` as dependency)

## 3. Sprint Replay

### Route: `/projects/$projectId/sprint/$sprintId/replay`

Replay recorded sprint events on a timeline. Reuses `ActivityFeed` and `FeedEvent` from v0.2.2.

### Layout

```
┌─── Replay Header ────────────────────────────────────────────┐
│ Sprint goal | Sprint # | Date | Duration                     │
├──────────────────────────────────────────────────────────────┤
│ ActivityFeed (shows events up to current playback position)  │
│                                                              │
│                                                              │
├─── Timeline Scrubber ────────────────────────────────────────┤
│ ◄ 00:00 ═══════●═══════════════════════════════════ 12:34 ► │
├─── Playback Controls ────────────────────────────────────────┤
│ |◄  ◄◄  ▶  ►►  ►|    Speed: [1x] [2x] [4x] [8x]           │
└──────────────────────────────────────────────────────────────┘
```

### Components

#### `SprintReplay.tsx`
Top-level component. Manages playback state, filters events by current time position.

```typescript
interface ReplayState {
  events: SprintEvent[];       // all events for the sprint
  currentTime: number;         // milliseconds from sprint start
  isPlaying: boolean;
  playbackSpeed: number;       // 1, 2, 4, 8
  duration: number;            // total replay duration in ms
}
```

Uses `useReplayEvents(sprintId)` (existing hook) to fetch all events. Converts `event.timestamp` to relative offset from first event.

Playback logic: `requestAnimationFrame` loop advances `currentTime` by `(elapsed * playbackSpeed)`. Events with `offset <= currentTime` are passed to `ActivityFeed`.

#### `TimelineScrubber.tsx`
Horizontal slider with event density markers.

```typescript
interface TimelineScrubberProps {
  currentTime: number;
  duration: number;
  eventMarkers: { offset: number; type: string }[];
  onSeek: (time: number) => void;
}
```

- Rendered as a styled `<input type="range">` with tick marks at event positions
- Colored markers: green (task complete), red (task failed), yellow (approval)
- Dragging pauses playback, releasing resumes

#### `PlaybackControls.tsx`
Transport buttons and speed selector.

```typescript
interface PlaybackControlsProps {
  isPlaying: boolean;
  speed: number;
  onPlay: () => void;
  onPause: () => void;
  onSpeedChange: (speed: number) => void;
  onSkipToStart: () => void;
  onSkipToEnd: () => void;
  onStepBack: () => void;    // jump to previous event
  onStepForward: () => void; // jump to next event
}
```

Keyboard shortcuts:
- `Space` — play/pause
- `←` / `→` — step back/forward one event
- `1`-`4` — set speed 1x/2x/4x/8x

## New API Endpoints

### Backend Additions

| Method | Endpoint | Returns | Purpose |
|--------|----------|---------|---------|
| GET | `/projects/{id}/analytics/cost-over-time` | `SprintCostPoint[]` | Cost per sprint over time |
| GET | `/projects/{id}/analytics/sprint-history` | `SprintSummary[]` | Tasks/completion per sprint |
| GET | `/projects/{id}/analytics/agent-performance` | `AgentMetrics[]` | Per-agent task/cost metrics |
| GET | `/analytics/global` | `GlobalAnalytics` | Aggregate stats across all projects |

### Existing Endpoints Used

| Endpoint | Used By |
|----------|---------|
| `GET /projects/{id}/cost` | AgentCostPieChart, StatCards |
| `GET /projects/{id}/learnings` | LearningsTimeline |
| `GET /sprints/{id}/replay` | SprintReplay |
| `GET /projects/{id}/sprints` | Sprint selector in replay |
