## 2. Polish & UX Improvements

### 2.1 Code-Splitting with React.lazy + Suspense

The Vite build currently produces oversized chunks due to Recharts and the replay module. Lazy-load heavy routes.

#### `router.tsx` Changes

Replace direct imports with lazy imports for heavy routes:

```typescript
import { lazy, Suspense } from 'react';
import { RouteLoadingFallback } from '@/components/shared/RouteLoadingFallback';

// Eager (lightweight)
import { DashboardPage } from '@/routes/index';
import { NewProjectPage } from '@/routes/projects/new';
import { ProjectDetailPage } from '@/routes/projects/$projectId/index';
import { SettingsPage } from '@/routes/settings';
import { TemplatesPage } from '@/routes/templates';

// Lazy (heavy dependencies: Recharts, CodeMirror, replay logic)
const AnalyticsPage = lazy(() =>
  import('@/routes/projects/$projectId/analytics').then(m => ({ default: m.AnalyticsPage }))
);
const SprintReplayPage = lazy(() =>
  import('@/routes/projects/$projectId/sprint/$sprintId/replay').then(m => ({ default: m.SprintReplayPage }))
);
const SprintCommandCenter = lazy(() =>
  import('@/routes/projects/$projectId/sprint').then(m => ({ default: m.SprintCommandCenter }))
);
const TeamConfigPage = lazy(() =>
  import('@/routes/projects/$projectId/team').then(m => ({ default: m.TeamConfigPage }))
);
```

Wrap lazy components in route definitions:

```typescript
// For each lazy route, wrap component with Suspense
component: () => (
  <Suspense fallback={<RouteLoadingFallback />}>
    <AnalyticsPage />
  </Suspense>
)
```

#### `RouteLoadingFallback.tsx`

```typescript
interface RouteLoadingFallbackProps {
  message?: string;
}
```

Full-page centered spinner with optional message. Uses the existing `LoadingSkeleton` pattern but simplified for route transitions.

### 2.3 Toast / Notification System

Use `sonner` library for toast notifications.

#### Setup

```typescript
// In AppShell.tsx or main.tsx
import { Toaster } from 'sonner';

// Add to layout:
<Toaster
  position="bottom-right"
  theme={theme}
  richColors
  closeButton
/>
```

#### `useToast.ts` Hook

Thin wrapper around `sonner` for consistent usage:

```typescript
import { toast } from 'sonner';

export function useToast() {
  return {
    success: (message: string) => toast.success(message),
    error: (message: string, description?: string) =>
      toast.error(message, { description }),
    info: (message: string) => toast.info(message),
    promise: <T>(promise: Promise<T>, messages: {
      loading: string;
      success: string;
      error: string;
    }) => toast.promise(promise, messages),
  };
}
```

#### Usage in Mutations

All existing mutation hooks should show toasts on success/error:

```typescript
// Example: useUpdateSettings
const mutation = useMutation({
  mutationFn: (data) => api.settings.update(data),
  onSuccess: () => {
    toast.success('Settings saved');
    queryClient.invalidateQueries({ queryKey: ['settings'] });
  },
  onError: (err) => toast.error('Failed to save settings', { description: err.message }),
});
```

Apply to all existing mutations:
- `useCreateProject` вҶ’ "Project created"
- `useCreateSprint` вҶ’ "Sprint created"
- `useApproveAction` вҶ’ "Approved"
- `useRejectAction` вҶ’ "Rejected"
- `useUpdateAgent` вҶ’ "Agent config updated"
- `useCreateTemplate` вҶ’ "Template created"
- `useDeleteTemplate` вҶ’ "Template deleted"
- `useApplyTemplate` вҶ’ "Template applied"

Each route component exports or computes its breadcrumbs:

#### Integration in `Header.tsx`

Add `<Breadcrumbs>` to the left side of the header, after the mobile menu button. On mobile, show only the last 2 breadcrumb items to save space.

```typescript
// Header.tsx updated layout:
<header className="h-14 border-b border-border flex items-center justify-between px-4 bg-card">

  </div>
  {/* ... right side unchanged */}

```

Breadcrumbs are passed via a Zustand store (`useUI`) or a context provider so child routes can set them:

```typescript
// In ui store, add:
breadcrumbs: BreadcrumbItem[];
setBreadcrumbs: (items: BreadcrumbItem[]) => void;
```

Each route calls `setBreadcrumbs(...)` in a `useEffect` on mount.

### 3.2 Focus Management

#### Route Transitions
On route change, move focus to `#main-content` heading or the first `<h1>` on the page:

```typescript
// In AppShell or a custom hook
const location = useLocation();
useEffect(() => {
  const heading = document.querySelector('#main-content h1, #main-content [role="heading"]');
  if (heading instanceof HTMLElement) {
    heading.focus();
    heading.setAttribute('tabindex', '-1');
  }
}, [location.pathname]);
```

#### Modal Focus Trapping
All modals (command palette, add API key, confirmation dialogs) must trap focus inside them. Use a `useFocusTrap` hook:

```typescript
function useFocusTrap(ref: React.RefObject<HTMLElement>, active: boolean): void;
```

Traps Tab/Shift+Tab within the ref element. Returns focus to trigger element on close.

## Theme System Enhancement

The current theme toggle in `Header.tsx` switches between `dark` and `light`. Extend to support `system`:

```typescript
// In ui store
type Theme = 'dark' | 'light' | 'system';

function applyTheme(theme: Theme) {
  const resolved = theme === 'system'
    ? (window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light')
    : theme;
  document.documentElement.classList.toggle('dark', resolved === 'dark');
  localStorage.setItem('foundrai-theme', theme);
}

// Listen for system changes when theme === 'system'
window.matchMedia('(prefers-color-scheme: dark)')
  .addEventListener('change', (e) => {
    if (getCurrentTheme() === 'system') {
      document.documentElement.classList.toggle('dark', e.matches);
    }
  });
```

The `AppearancePanel` in Settings provides the three-option selector. The `Header.tsx` toggle cycles through: dark вҶ’ light вҶ’ system вҶ’ dark.
